Project 2: Election Night

Submission deadline: 6pm, Friday 1 June 2012.
Value: 15% of CITS1401.
To be done in pairs. You are free to find your own partner for the project. Only one submission is necessary, but make sure that both names are clearly stated.

You should construct a Python program containing your solution to the following problem. You must submit your program electronically using cssubmit. No other method of submission is allowed.

You are expected to have read and understood the University's guidelines on academic conduct. In accordance with this policy, you may discuss with other students the general principles required to understand this project, but the work you submit must be the result of your own effort.

You must submit your project before the submission deadline above. The penalty for late submission is described in the unit outline.

Overview

The Government of Antarctica conducts its elections using a variant of optional preferential voting. Each constituency elects one member to the Antarctican Parliament, and each voter casts one vote in one of three forms.

  • A ranked vote: the voter can use the numbers 1, 2, ..., n to indicate preferences amongst the n candidates - but note that they are not required to rank all of the candidates.
  • A marked vote: the voter can simply place a mark (any mark) next to their preferred candidate. The rest of the paper must be blank. This is equivalent to placing a 1 next to that candidate.
  • A written vote: the voter can delegate their vote to their preferred candidate by writing the candidate's name on the paper, in which case their vote is replaced by the candidate's pre-registered voting ticket. A pre-registered voting ticket is always a ranked vote. If the chosen candidate has not registered a voting ticket, the vote is interpreted as a simple mark against the candidate's name.
No more than nine candidates are allowed in any one constituency. Any ballot paper that does not fit the above description is declared informal and is disregarded in the count. Example ballot papers and their interpretations are provided.

Once each ballot paper has been interpreted, the election count proceeds exactly as in an Australian Federal Lower House election. With v formal votes, the winning line is set initially at v//2 + 1 votes.

  • Each vote is allocated to the candidate who is its first preference.
  • If the leader in the count is over the winning line, they are declared the winner and the count is complete. Otherwise, the trailing candidate (the one with the least votes) is identified, they are eliminated from the count, and each of their votes is re-distributed to its highest-ranked surviving candidate. Note that some of the trailing candidate's votes may expire at this stage (if the voter did not rank all of the candidates), so the winning line may need to be adjusted. You may assume that there is never a tie for the trailing candidate.

In this project you will write a Python program that reads in a file of candidates and their voting tickets (example here), plus a file of voters' ballot papers (example here), and that conducts and illustrates the counting process for that election (results from the examples here).

Tasks

You are required to write the following Python functions. Make sure you understand where each function fits into the process described above.

  1. def lines(f):
    lines(f) takes the name of a file, and it returns a list of strings, each holding one line from the file. lines(f) returns [] if f doesn't exist.

  2. def candidates(f):
    candidates(f) takes the name of a file describing the candidates in an election, and it returns a list of tuples, each containing the name of a candidate and their pre-registered voting ticket (or an empty string if there is no ticket).
    e.g. given file.txt, candidates("file.txt") returns [("AB", "132"), ("C D", ""), ("EFG", ""), ("HJ K", "2 1")].
    You may assume that the format of the file is always correct, and you should disregard any blank lines in the file.

  3. def rankedVote(p, cs):
    rankedVote(p, cs) takes a string p and a list of tuples cs that describes the candidates in an election, and it returns a list of strings holding the interpretation of p as a ranked vote.
    e.g. given cs = [("AB", "132"), ("C D", ""), ("EFG", ""), ("HJ K", "2 1")]:
    rankedVote("21 3", cs) returns ["C D", "AB", "HJ K"],
    rankedVote("2 12", cs) returns ["EFG"], and
    rankedVote(p, cs) returns [] for all p which aren't valid ranked votes. See formal.html for more examples.

  4. def zeroElection(cs):
    zeroElection(cs) takes a list of tuples cs that describes the candidates in an election, and it returns a dictionary containing one entry for each candidate. Each entry is a tuple of the form (0, [], k, x), where k is a distinct index from 0 for each candidate, and x is the vote corresponding to the candidate's pre-registered voting ticket.
    e.g. zeroElection([("AB", "132"), ("C D", ""), ("EFG", ""), ("HJ K", "2 1")]) returns the dictionary
    {"AB": (0, [], 0, ["AB", "EFG", "C D"]), "C D": (0, [], 1, ["C D"]), "EFG": (0, [], 2, ["EFG"]), "HJ K": (0, [], 3, ["HJ K", "AB"])}.

    In each tuple, the second field holds the current list of votes for that candidate, and the first field always holds the length of that list. We shall refer to a dictionary of this form as an election status.

  5. def markedVote(p, cs):
    markedVote(p, cs) takes a string p and a list of tuples cs that describes the candidates in an election, and it returns a list of strings holding the interpretation of p as a marked vote.
    e.g. given cs = [("AB", "132"), ("C D", ""), ("EFG", ""), ("HJ K", "2 1")]:
    markedVote(" x", cs) returns ["C D"],
    markedVote("2 ", cs) returns ["AB"], and
    markedVote(p, cs) returns [] for all p which aren't valid marked votes. See formal.html for more examples.

  6. def writtenVote(p, piles):
    writtenVote(p, piles) takes a string p and an election status piles, and it returns a list of strings holding the interpretation of p as a written vote wrt piles. writtenVote(p, piles) returns [] for all p which aren't valid written votes. See formal.html for more examples.

  7. def paperToVote(p, cs, piles):
    paperToVote(p, cs, piles) takes a string p, a list of tuples cs that describes the candidates in an election, and a corresponding election status piles, and it returns a list of strings holding the interpretation of p as a vote. paperToVote(p, cs, piles) returns [] for all p which aren't valid votes.

  8. def distributeVotes(vs, piles):
    distributeVotes(vs, piles) takes a list of votes vs and an election status piles, and it returns a tuple containing the new piles where each vote in vs has been distributed to its highest-ranked surviving candidate, and the number of empty votes from vs.

    Note that if a vote v is distributed to a candidate c, then all entries on v up to and including c should be deleted.
    e.g. if ["A", "B", "C", "D"] is distributed to C (presumably because "A" and "B" have already been eliminated from the election), then the vote added to C's pile should be just ["D"].

  9. def leader(piles), piles not empty:
    leader(piles) takes an election status piles, and it returns the name of the candidate who has the most votes.

  10. def loser(piles), piles not empty:
    loser(piles) takes an election status piles, and it returns the name of the candidate who has the least votes.

  11. def nextRound(piles), piles not empty:
    nextRound(piles) takes an election status piles, it identifies the trailing candidate c, and it returns a tuple containing
    • piles after c has been removed and c's votes have been re-distributed, and
    • the number of c's votes that expired.
    The re-distributed votes should be cut back as described under distributeVotes.

  12. def displayStandings(piles, ...), piles not empty:
    displayStandings(piles, ...) takes an election status piles, and it uses turtle graphics to display the current standings on the screen. results.txt lists the kind of information that your display should include.

    If you want to pass other arguments to displayStandings, that's fine: it will not be called directly by the testing or marking programs.

  13. def main():
    main() prompts the user for the names of a file of candidates' information and a file of completed ballot papers, and it conducts an Antarctican election and displays the results on the screen.

Submission

Submit a single file antarctica.py containing all of your functions via cssubmit.

Your file name and your function names and types must match the specifications exactly. The marking process is partly automated and any deviation in the names will cause problems, which will be penalised.

Assessment

Your submission will be assessed on

  • completeness: how many of the functions you have submitted;
  • correctness: whether your functions implement the specifications exactly;
  • presentation: how well your program illustrates the counting process;
  • clarity: whether your code is clear to read and well-constructed.

A testing program is provided for you to validate your program before submission. Note however that the correctness of your program is your responsibility: a program that passes all of the tests in the testing program is not guaranteed to be free of errors. "Testing shows the presence, not the absence of bugs." -- E.W. Dijkstra.

Also compare your program's output with the sample results, generated from the sample candidates and sample papers files.

Prize

There is a prize for the best election program submitted in the usual way before the project deadline.

Lyndon retains full discretion over the awarding of all prizes. Submit only original programs written in Python that use turtle graphics.
